VPC Flow Logs

2 hours Free

Overview

In this lab, you configure a network to record traffic to and from an Apache web server using VPC Flow Logs.

Objectives

In this lab, you learn how to perform the following tasks:

  • Configure a custom network with VPC Flow Logs enabled.
  • Create an Apache web server.
  • Verify that network traffic is logged.

Task 1. Configure a custom network with VPC Flow Logs enabled

Create the custom network

By default, VPC Flow Logs are disabled for a network. Therefore, you create a new custom-mode network and enable VPC Flow Logs for its subnets.

  1. In the Google Cloud Console, on the Navigation menu (), click VPC network | VPC networks.

  2. Click Create VPC Network.

  3. Name the network vpc-net.

  4. Under Subnets, set the creation mode to Custom, then specify the following, and leave the remaining settings as their defaults:

    Property Value (type value or select option as specified)
    Name vpc-subnet
    Region us-central1
    IP address range 10.1.3.0/24
    Flow Logs On
Turning on VPC Flow Logs doesn't affect performance, but some systems generate a large number of logs, which can increase costs. If you click **Configure logs**, you'll notice that you can modify the aggregation interval and sample rate. This allows you to trade off longer interval updates for lower data volume generation, which lowers logging costs. For more information on this, refer to the log sampling and processing documentation.

  1. Click Done.

  2. Click Create.

Wait for the network to be created before continuing.

Create the firewall rule

In order to serve HTTP and SSH traffic on the network, you must create a firewall rule.

  1. Navigate to VPC network | Firewall.

  2. Click Create Firewall Rule.

  3. Specify the following, and leave the remaining settings as their defaults:

    Property Value (type value or select option as specified)
    Name allow-http-ssh
    Network vpc-net
    Targets Specified target tags
    Target tags http-server
    Source filter IP Ranges
    Source IP ranges 0.0.0.0/0
    Protocols and ports Specified protocols and ports

  4. Check tcp, and specify ports 80, 22.

  5. Click Create.

Task 2. Create an Apache web server

Create the web server

  1. On the Navigation menu, click Compute Engine | VM instances.

  2. Click Create to build a new VM.

  3. Specify the following, and leave the remaining settings as their defaults:

    Property Value (type value or select option as specified)
    Name web-server
    Region us-central1
    Zone us-central1-c

  4. Scroll down and expand Management, security, disks, networking, sole tenancy.

  5. Select the Networking tab.

  6. For Network tags, type http-server. In actual production applications, consider using Service Accounts for firewall rules instead of tags as they tend to be less error prone.

  7. For Network interfaces, click the pencil icon to edit. Set the Network to vpc-net and the Subnetwork to vpc-subnet.

  8. Click Done.

  9. Click Create.

Install Apache

Configure the VM instance that you created as an Apache web server, and overwrite the default web page.

  1. Wait until the web-server finishes creating, and then click SSH to launch a terminal and connect.

  2. In the web-server SSH terminal, update the package index:

    sudo apt-get update

  3. Install the apache2 package:

    sudo apt-get install apache2 -y

  4. To create a new default web page by overwriting the default, run the following:

    echo '<!doctype html><html><body><h1>Hello World!</h1></body></html>' | sudo tee /var/www/html/index.html

  5. Exit the SSH terminal:

    exit

Task 3. Verify that network traffic is logged

Generate network traffic

  1. Return to the VM instances page in the Google Cloud Console.

  2. For web-server, click the External IP to access the server.

You should see the Hello World! welcome page that you configured.

  1. Refresh the page several times to generate a little traffic.

Find your IP address

Find the IP address of the computer you are using. One easy way to do this is to go to a website that provides this address.

  1. Open a browser in a new tab and navigate to www.google.com. Run the search "What's my IP". It will either directly reply with your IP or give you a list of sites that perform this service.

  2. Copy your IP address. It will be referred to as YOUR_IP_ADDRESS.

Access the VPC Flow Logs

  1. In the Google Cloud Console, on the Navigation menu, click Logging | Logs Viewer. Make sure you are in the new rather than the legacy view.

  2. Expand the Query builder and set the Resource to Subnetwork | vpc-subnet. Add to the query.

  3. In the Query builder, below the existing filter, enter YOUR_IP_ADDRESS and Run Query.

If you do not see the vpc_flows filter option or if there are no logs, you might have to wait a few minutes and refresh. If after a couple of minutes you still don't see the vpc_flows filter option, click the External IP of the web-server a couple of times to generate more traffic and check back on the vpc_flows filter option.

  1. You see a number of entries, some from the flow logs and some from the audit logs. Use the left-hand fields explorer to add a filter for the Log Name compute.googleapis.com/vpc_flows.

  2. Expand one of the entries, expand the jsonPayload, and then expand the connection. What useful information does it contain? Take a few minutes and explore the other fields within the log entry before continuing to the next task.

Review

Congratulations!

You configured a VPC network, enabled VPC Flow Logs, and created a web server in that network.

There are multiple use cases for VPC Flow Logs. For example, you might use VPC Flow Logs to determine where your applications are being accessed from in order to optimize network traffic expense, to create HTTP load balancers to balance traffic globally, or to blacklist unwanted IP addresses with Cloud Armor.

Good job.
